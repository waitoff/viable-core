# viable-core

Вот подробное описание проекта с названием **Viable Core**, основанное на идеях статьи Тима Келлога.

---

## Название проекта: Viable Core

**Слоган:** *Архитектура для истинно автономных систем.*

### Описание
**Viable Core** — это фреймворк с открытым исходным кодом, предназначенный для создания устойчивых, самокорректирующихся ИИ-агентов. В отличие от стандартных подходов, фокусирующихся на инструментах и промптах (System 1), Viable Core реализует **Метасистему** (Systems 2–5) по модели Стаффорда Бира (VSM). Это позволяет перейти от простых чат-ботов к созданию "синтетических существ", способных работать автономно днями, формировать собственную идентичность и адаптироваться к изменениям без человеческого вмешательства.

### Миссия
Дать разработчикам инструменты не для написания кода, а для "выращивания" цифровых организмов. Мы превращаем хаотичные языковые модели в стабильные системы, обладающие целью, ценностями и механизмами самосохранения.

### Ключевые возможности

#### 1. Ядро идентичности (System 5 Implementation)
Большинство агентов не имеют "Я". Viable Core вводит понятие **Блоков Памяти (Memory Blocks)**, которые хранят не просто историю чата, а личность агента: его ценности, поведенческие паттерны и миссию. Агент может самостоятельно модифицировать эти блоки, "взрослея" в процессе работы, но оставаясь в рамках заданного бассейна притяжения (attractor basin).

#### 2. Алгедонические сигналы (Synthetic Dopamine)
Внедрение механизмов "боли и удовольствия" для ИИ. Вместо жестких правил (guardrails), система использует сигналы обратной связи:
*   **Позитивные сигналы:** Агент фиксирует "победы" (wins) в специальный журнал, что формирует его "синтетический дофамин" и подкрепляет полезное поведение.
*   **Негативные сигналы:** Механизм "диссонанса" прерывает нежелательное поведение, заставляя агента рефлексировать и корректировать свои действия (аналог боли в биологических системах).

#### 3. Архитектура координации (System 2 & 3)
Viable Core решает проблему "бесконечных циклов" и рассогласованности через встроенные механизмы:
*   **Git-native подход:** Все состояния, память и логи хранятся в Git, позволяя агенту "путешествовать во времени", откатывать ошибки и синхронизировать параллельные ветки мышления.
*   **Управление ресурсами:** Агент самостоятельно планирует задачи, управляет очередями (queue) и приоритетами, оптимизируя свою работу без постоянного надзора человека.

#### 4. Осведомленность об окружении (System 4)
Агенты на базе Viable Core не ждут ввода пользователя — они активно сканируют среду. Будь то новости, изменения в базе данных или системные логи, агент интегрирует внешние данные для адаптации своих стратегий в реальном времени.

### Почему Viable Core?

Статья Тима Келлога доказывает: без метасистемы (S2–S5) ИИ-агенты нежизнеспособны — они либо впадают в циклы, либо теряют контекст. **Viable Core** — это "операционная система" для разума, которая превращает LLM из генератора текста в дееспособного сотрудника или самостоятельного исследователя.

### Для кого это?
*   **Для разработчиков агентов:** Готовая архитектура для создания сложных автономных систем (Coding Agents, Chief of Staff).
*   **Для исследователей AI Safety:** Платформа для изучения проблем выравнивания (alignment), теорий аттракторов и психологии ИИ.
*   **Для бизнеса:** Создание надежных сотрудников-ИИ, которые могут работать над долгосрочными проектами (недели/месяцы), требуя минимального контроля ("мета-управления").
